# Sleep mode and auto-delete for virtual clusters

:::note
Platform sleep has been integrated into the [standard sleep mode configuration](/vcluster/configure/vcluster-yaml/sleep-mode) and is no longer a separate feature.
:::

Our platform provides two features to reduce Kubernetes costs:

- **Sleep mode** which puts virtual clusters to sleep when nobody is using them, i.e. purging all pods while keeping all resources inside the virtual clusters during periods of inactivity.
- **Auto-delete** which deletes virtual clusters that have been idle for a while.

Both of these feature rely on vCluster Platform's [inactivity detection](#inactivity-detection).
