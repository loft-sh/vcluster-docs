ClusterRoleTemplate holds the clusterRoleTemplate information

## ClusterRoleTemplate example

An example ClusterRoleTemplate:
```yaml
apiVersion: management.loft.sh/v1
kind: ClusterRoleTemplate
metadata:
  creationTimestamp: null
spec:
  access:
  - users:
    - '*'
    verbs:
    - get
  clusterRoleTemplate:
    metadata:
      creationTimestamp: null
    rules:
    - apiGroups:
      - management.loft.sh/v1
      resources:
      - spaceinstances
      - virtualclusterinstances
      verbs:
      - get
      - list
      - update
    - apiGroups:
      - management.loft.sh/v1
      resources:
      - projectsecrets
      verbs:
      - get
      - list
  description: |-
    Allows the user or team to manage the project. Gives only access to existing
        space and virtual cluster objects but no creation or deletion privileges.
  displayName: Project User -No Create
  management: true
status: {}

```
