This procedure describes how to upgrade vCluster Platform with minimal downtime.
Because all replicas run in a single cluster, a standard rolling upgrade handles
the transition automatically.

## Prerequisites

- Updated values file (`platform-ha-values.yaml`)
- `kubectl` context configured for the cluster
- The target chart version available in the Helm repository

## Step 1 - Upgrade the deployment

<Tabs
  defaultValue="cli"
  values={[
    { label: "vCluster CLI", value: "cli" },
    { label: "Helm", value: "helm" },
  ]}
>
  <TabItem value="cli">

<InterpolatedCodeBlock
  code={
`vcluster platform start \\
  --namespace vcluster-platform \\
  --kube-context [[GLOBAL:CLUSTER_CONTEXT]] \\
  --values platform-ha-values.yaml \\
  --upgrade \\
  --no-tunnel`
  }
  language="bash"
/>

  </TabItem>
  <TabItem value="helm">

<InterpolatedCodeBlock
  code={
`helm upgrade loft vcluster-platform --install --create-namespace --repository-config='' \\
  --namespace vcluster-platform \\
  --repo "https://charts.loft.sh/" \\
  --version [[VAR:CHART_VERSION:4.8.0]] \\
  --kube-context [[GLOBAL:CLUSTER_CONTEXT]] \\
  -f platform-ha-values.yaml \\
  --server-side=true --force-conflicts`
  }
  language="bash"
/>

The `--server-side=true --force-conflicts` flags are required because the platform chart manages CRDs that may conflict during upgrades.

  </TabItem>
</Tabs>

## Step 2 - Wait for the rollout to complete

<InterpolatedCodeBlock
  code={
`kubectl --context [[GLOBAL:CLUSTER_CONTEXT]] \\
  rollout status deployment/loft -n vcluster-platform`
  }
  language="bash"
/>

## Step 3 - Verify the deployment

Confirm that all platform and agent deployments are running the expected image:

<InterpolatedCodeBlock
  code={
`echo -n "Platform: " && kubectl --context [[GLOBAL:CLUSTER_CONTEXT]] get deployment -n vcluster-platform loft \\
  -o jsonpath='{.spec.template.spec.containers[0].image}' && echo`
  }
  language="bash"
/>

The platform automatically upgrades connected agents after the platform upgrade
completes.
