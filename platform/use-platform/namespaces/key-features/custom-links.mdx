You can add custom links to namespace instances, for example to point to a GitHub pull request or Jira issue.

:::tip Virtual cluster custom links
To learn how to configure custom links, see [Advanced Topics](../../virtual-clusters/key-features/custom-links.mdx) in the Virtual Clusters section.
:::

<Tabs
  defaultValue="ui"
  values={[
    { label: 'UI', value: 'ui' },

  <TabItem value="ui">
    <Steps>
      <Step>
        Go to the <NavStep>Namespaces</NavStep> view using the menu on the left
      </Step>
      <Step>
        Edit an existing Namespace or create a new one
      </Step>
      <Step>
        Click <Label>Add Link</Label> to add a new link or <Label>Change</Label> to edit existing ones
      </Step>
      <Step>
        Enter a URL and optionally a label for your link
      </Step>
      <Step>
        Save your links by clicking <Label>Save</Label>
      </Step>
      <Step>
        Click on the <Button>Create Namespace</Button>,
        or <Button>Save Changes</Button> if you're editing an existing namespace,
        to apply all changes
      </Step>
    </Steps>
  </TabItem>
  <TabItem value="cli">

```bash
vcluster platform create namespace namespace-1 --link 'my-issue=https://github.com/kubernetes/kubernetes/pull/1234'
vcluster platform create namespace namespace-1 --link 'https://loft.sh'
vcluster platform create namespace namespace-1 --link 'my-issue=https://github.com/kubernetes/kubernetes/pull/1234,documentation=https://example.com/docs'
```
</TabItem>
