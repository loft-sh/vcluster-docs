Virtual clusters created through the vcluster CLI can be imported into a project. This process will
automatically create a `VirtualClusterInstance` resource in the specified project.

Importing virtual clusters can be accomplished via the UI or the CLI.

<Tabs
  defaultValue="ui"
  values={[
    { label: "UI", value: "ui" },
    { label: "CLI", value: "cli" },
  ]}
>
  <TabItem value="ui">
    <Flow id="import-vcluster">
      <Step>
        Click on <NavStep>Clusters</NavStep> in the side bar to expand the sidebar. 
      </Step>
      <Step>
        Click on <NavStep>Clusters</NavStep> to see the list of clusters. 
      </Step>
      <Step>
        Click on the cluster containing the virtual cluster that you would like to import.
      </Step>
      <Step>
        In the Cluster Management view, click the
        <Label>Virtual Clusters</Label> to view the list of virtual clusters.
      </Step>
      <Step>
        Find the virtual cluster you would like to import in the list of virtual
        clusters. Click the drop down arrow next to the virtual cluster name. In
        the drop down menu click the <Label>Import</Label> button.
      </Step>
      <Step>
        In the pop-up box, select the project you would like to import the
        virtual cluster into from the <Label>Select Project</Label> drop down
        box.
      </Step>
      <Step>
        <b>[Optional]</b> provide a name for the new 'VirtualClusterInstance'
        resource, if you do not provide a name, vCluster Platform will use the name
        of the legacy virtual cluster.
      </Step>
      <Step>
        Click the <Button>Import</Button> button.
      </Step>
    </Flow>
  </TabItem>
