The observability gateway runs in the vCluster Platform namespace. It contains two
containers:

- `otel-gateway`, the Write Gateway that receives OTLP metrics.
- `query-proxy`, the Prometheus-compatible read proxy.

Platform deploys the gateway from an observability connector Secret. The connector
stores the metrics backend endpoints and optional TLS settings.

## Prerequisites

Before you create the connector, prepare:

- A metrics backend that accepts OTLP metrics from the gateway.
- A Prometheus-compatible query endpoint for the same backend.
- Network access from the Platform namespace to both backend endpoints.
- Optional CA, gateway serving certificate, or backend client certificate Secrets.

:::note
Phase 1 does not install or size the metrics backend for you. Use your existing backend
or deploy one separately.
:::

## Create the connector Secret

Create one fleet observability connector in the Platform namespace. The fleet connector
must use the name `fleet-observability` and the label
`loft.sh/fleet-observability-connector: "true"`.

```yaml title="fleet-observability-connector.yaml"
apiVersion: v1
kind: Secret
metadata:
  name: fleet-observability
  namespace: vcluster-platform
  labels:
    loft.sh/connector-type: observability
    loft.sh/fleet-observability-connector: "true"
type: Opaque
stringData:
  metricsBackendOtlpEndpoint: https://metrics.example.com/otlp
  metricsBackendPromQLEndpoint: https://metrics.example.com
```

Apply the Secret:

```bash
kubectl apply -f fleet-observability-connector.yaml
```

Platform reconciles the following resources:

- `Deployment/gateway-fleet-observability`
- `Service/gateway-fleet-observability`
- `ServiceAccount/gateway-fleet-observability`
- `ConfigMap/gateway-fleet-observability`
- `ClusterRole/gateway-fleet-observability`
- `ClusterRoleBinding/gateway-fleet-observability`
- Grafana datasource ConfigMaps for fleet, Control Plane Cluster, and tenant cluster
  views when Grafana datasource reconciliation is enabled.

## Connector keys

| Key | Required | Description |
|---|---:|---|
| `metricsBackendOtlpEndpoint` | Yes | OTLP endpoint that receives metrics from the Write Gateway. |
| `metricsBackendPromQLEndpoint` | Yes | Prometheus HTTP API endpoint queried by the Query Proxy. |
| `metricsBackendCaData` | No | PEM CA data used to verify the metrics backend. |
| `metricsBackendInsecureSkipVerify` | No | Set to `"true"` only for test backends with unverifiable TLS. |
| `metricsBackendServerName` | No | TLS server name override for the metrics backend. |
| `observabilityCaSecretName` | No | Secret name with `ca.crt` used to verify the backend and gateway datasource TLS. |
| `observabilityGatewayCertSecretName` | No | Secret name with `tls.crt` and `tls.key` for gateway serving TLS. |
| `metricsBackendCertSecretName` | No | Secret name with `tls.crt` and `tls.key` for mTLS to the backend. |
| `gatewayResources` | No | YAML `resources` block for the Write Gateway container. |
| `queryProxyResources` | No | YAML `resources` block for the Query Proxy container. |

If both `metricsBackendCaData` and `observabilityCaSecretName` are set, the Secret
referenced by `observabilityCaSecretName` is used.

## Enable gateway TLS

To serve OTLP and query traffic over TLS, create a Secret in the Platform namespace:

```yaml title="gateway-tls.yaml"
apiVersion: v1
kind: Secret
metadata:
  name: fleet-gateway-tls
  namespace: vcluster-platform
type: kubernetes.io/tls
stringData:
  tls.crt: |
    -----BEGIN CERTIFICATE-----
    ...
    -----END CERTIFICATE-----
  tls.key: |
    -----BEGIN PRIVATE KEY-----
    ...
    -----END PRIVATE KEY-----
```

Reference it from the connector:

```yaml
stringData:
  observabilityGatewayCertSecretName: fleet-gateway-tls
```

Edge collectors must then use `https` for OTLP HTTP or TLS for OTLP gRPC. Grafana
datasources generated by Platform use the same query proxy Service and include the
CA data when you also configure `observabilityCaSecretName`.

## Use backend mTLS

To authenticate the gateway to your backend with mTLS, create a Secret:

```yaml title="backend-client-tls.yaml"
apiVersion: v1
kind: Secret
metadata:
  name: metrics-backend-client-tls
  namespace: vcluster-platform
type: kubernetes.io/tls
stringData:
  tls.crt: |
    -----BEGIN CERTIFICATE-----
    ...
    -----END CERTIFICATE-----
  tls.key: |
    -----BEGIN PRIVATE KEY-----
    ...
    -----END PRIVATE KEY-----
```

Reference it from the connector:

```yaml
stringData:
  metricsBackendCertSecretName: metrics-backend-client-tls
```

## Verify the gateway

Check that the gateway pod is running:

```bash
kubectl get pods -n vcluster-platform -l app.kubernetes.io/name=gateway,app.kubernetes.io/instance=fleet-observability
```

Check the service ports:

```bash
kubectl get svc -n vcluster-platform gateway-fleet-observability
```

Expected service ports:

| Port | Name | Purpose |
|---:|---|---|
| `4317` | `otlp` | OTLP gRPC writes. |
| `4318` | `otlp-http` | OTLP HTTP writes. |
| `8081` | `query` | Prometheus HTTP API through the Query Proxy. |

Next, create metrics access keys in [Configure metrics access](./configure-metrics-access.mdx).
