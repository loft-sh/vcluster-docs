# Annotations and labels reference {#annotations-reference}

This page documents the well-known annotations and labels in the `loft.sh`, `vcluster.loft.sh`, and `sleepmode.loft.sh` namespaces used by vCluster and vCluster Platform.

## Cluster annotations {#cluster-annotations}

These annotations are used on `Cluster` resources to configure cluster-level behavior.

### loft.sh/agent-values {#loft-sh-agent-values}

**Type:** Annotation

**Example:** `loft.sh/agent-values: "replicaCount: 2"`

**Used on:** Cluster

Provides custom Helm values for the vCluster Platform agent installed on the cluster. These values are merged with the default agent chart values and any platform-level `agentValues` configuration.

### loft.sh/cluster-ignore-agent {#loft-sh-cluster-ignore-agent}

**Type:** Annotation

**Example:** `loft.sh/cluster-ignore-agent: "true"`

**Used on:** Cluster

When set to `true`, prevents vCluster Platform from automatically managing agent updates on this cluster. Use this when you want to manually control agent versions.

### loft.sh/direct-cluster-endpoint {#loft-sh-direct-cluster-endpoint}

**Type:** Annotation

**Example:** `loft.sh/direct-cluster-endpoint: "cluster.example.com"`

**Used on:** Cluster

Specifies a direct endpoint for the cluster, enabling clients to connect directly to the cluster instead of routing through the central vCluster Platform instance. This is useful in multi-region deployments where direct connectivity reduces latency.

### loft.sh/direct-cluster-endpoint-insecure {#loft-sh-direct-cluster-endpoint-insecure}

**Type:** Annotation

**Example:** `loft.sh/direct-cluster-endpoint-insecure: "true"`

**Used on:** Cluster

When set to `true`, creates Kubernetes contexts with `insecure-skip-tls-verify: true` for the direct cluster endpoint. Use this when the endpoint uses self-signed certificates.

### loft.sh/direct-cluster-endpoint-ca-data {#loft-sh-direct-cluster-endpoint-ca-data}

**Type:** Annotation

**Example:** `loft.sh/direct-cluster-endpoint-ca-data: "LS0tLS1CRUdJTi..."`

**Used on:** Cluster

Contains the base64-encoded CA certificate for the direct cluster endpoint. Used when the endpoint uses a custom certificate authority.

### loft.sh/derp-endpoint {#loft-sh-derp-endpoint}

**Type:** Annotation

**Example:** `loft.sh/derp-endpoint: "eu-west-cluster.example.com"`

**Used on:** Cluster

Specifies a publicly accessible DERP relay endpoint for this cluster. DERP (Designated Encrypted Relay for Packets) enables connectivity between agents when direct connections are not possible.

### loft.sh/derp-endpoint-insecure {#loft-sh-derp-endpoint-insecure}

**Type:** Annotation

**Example:** `loft.sh/derp-endpoint-insecure: "true"`

**Used on:** Cluster

When set to `true`, allows insecure connections to the DERP relay endpoint. Use this for self-signed certificates. Default: `false`.

### loft.sh/ingress-suffix {#loft-sh-ingress-suffix}

**Type:** Annotation

**Example:** `loft.sh/ingress-suffix: "vclusters.example.com"`

**Used on:** Cluster

Sets the domain suffix used for vCluster ingress access points on this cluster. When configured, vCluster instances can be accessed directly via ingress instead of through the platform proxy.

## vCluster instance annotations {#vcluster-instance-annotations}

These annotations are used on vCluster instances and related resources.

### loft.sh/vcluster-instance-name {#loft-sh-vcluster-instance-name}

**Type:** Annotation

**Example:** `loft.sh/vcluster-instance-name: "dev-vcluster"`

**Used on:** Namespace, Pod

Identifies the vCluster instance name that owns or manages this resource.

### loft.sh/vcluster-instance-namespace {#loft-sh-vcluster-instance-namespace}

**Type:** Annotation

**Example:** `loft.sh/vcluster-instance-namespace: "loft-p-default"`

**Used on:** Namespace, Pod

Specifies the parent namespace in which the vCluster instance is deployed.

### loft.sh/vcluster-instance-project {#loft-sh-vcluster-instance-project}

**Type:** Annotation

**Example:** `loft.sh/vcluster-instance-project: "default"`

**Used on:** Namespace, Pod

Defines the vCluster Platform project to which this vCluster instance belongs.

### vcluster.loft.sh/name {#vcluster-loft-sh-name}

**Type:** Label

**Example:** `vcluster.loft.sh/name: "my-vcluster"`

**Used on:** All synced resources

Identifies the name of the vCluster that owns or synced this resource.

### vcluster.loft.sh/namespace {#vcluster-loft-sh-namespace}

**Type:** Label

**Example:** `vcluster.loft.sh/namespace: "vcluster-my-vcluster"`

**Used on:** All synced resources

Identifies the namespace where the vCluster is deployed in the host cluster.

### vcluster.loft.sh/managed-by {#vcluster-loft-sh-managed-by}

**Type:** Label

**Example:** `vcluster.loft.sh/managed-by: "vcluster"`

**Used on:** Synced resources

Indicates that this resource is managed by vCluster and should not be modified directly.

## Sync control annotations {#sync-control-annotations}

These annotations control how vCluster syncs resources between the virtual and host clusters.

### vcluster.loft.sh/force-sync {#vcluster-loft-sh-force-sync}

**Type:** Annotation

**Example:** `vcluster.loft.sh/force-sync: "true"`

**Used on:** Any resource

Forces vCluster to sync this resource from the host cluster to the virtual cluster, even if it would normally be excluded by sync configuration.

### vcluster.loft.sh/skip-translate {#vcluster-loft-sh-skip-translate}

**Type:** Annotation

**Example:** `vcluster.loft.sh/skip-translate: "true"`

**Used on:** Any resource

Prevents vCluster from translating (name-mangling) this resource when syncing. The resource retains its original name in the host cluster.

### vcluster.loft.sh/skip-backsync {#vcluster-loft-sh-skip-backsync}

**Type:** Annotation

**Example:** `vcluster.loft.sh/skip-backsync: "true"`

**Used on:** Synced resources

Prevents changes to this resource in the host cluster from being synced back to the virtual cluster.

### vcluster.loft.sh/import {#vcluster-loft-sh-import}

**Type:** Annotation

**Example:** `vcluster.loft.sh/import: "true"`

**Used on:** Resources in host cluster

Marks a resource in the host cluster for import into the vCluster. The resource will be synced into the virtual cluster.

## Sleep mode annotations {#sleep-mode-annotations}

These annotations control the sleep mode behavior for namespaces and vCluster instances.

### sleepmode.loft.sh/sleep-after {#sleepmode-loft-sh-sleep-after}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/sleep-after: "3600"`

**Used on:** Namespace, VirtualClusterInstance

Specifies the number of seconds of inactivity after which the namespace or vCluster should automatically sleep.

### sleepmode.loft.sh/delete-after {#sleepmode-loft-sh-delete-after}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/delete-after: "86400"`

**Used on:** Namespace, VirtualClusterInstance

Specifies the number of seconds of inactivity after which the namespace or vCluster should be automatically deleted.

### sleepmode.loft.sh/sleep-schedule {#sleepmode-loft-sh-sleep-schedule}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/sleep-schedule: "0 20 * * *"`

**Used on:** Namespace, VirtualClusterInstance

Specifies a cron schedule for when the namespace or vCluster should automatically sleep. Uses standard cron format.

### sleepmode.loft.sh/wakeup-schedule {#sleepmode-loft-sh-wakeup-schedule}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/wakeup-schedule: "0 8 * * 1-5"`

**Used on:** Namespace, VirtualClusterInstance

Specifies a cron schedule for when the namespace or vCluster should automatically wake up.

### sleepmode.loft.sh/timezone {#sleepmode-loft-sh-timezone}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/timezone: "America/New_York"`

**Used on:** Namespace, VirtualClusterInstance

Specifies the timezone for scheduled sleep and wakeup operations. Accepts IANA timezone names. Defaults to UTC.

### sleepmode.loft.sh/force {#sleepmode-loft-sh-force}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/force: "true"`

**Used on:** Namespace, VirtualClusterInstance

Forces the namespace or vCluster to sleep immediately, regardless of activity.

### sleepmode.loft.sh/force-duration {#sleepmode-loft-sh-force-duration}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/force-duration: "3600"`

**Used on:** Namespace, VirtualClusterInstance

Forces sleep for a specific duration in seconds. After this period, normal activity tracking resumes. Set to `0` for indefinite sleep until manually woken.

### sleepmode.loft.sh/exclude {#sleepmode-loft-sh-exclude}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/exclude: "true"`

**Used on:** Deployment, StatefulSet, ReplicaSet, Pod

Excludes this workload from sleep mode. When the namespace sleeps, this workload continues running.

### sleepmode.loft.sh/ignore-all {#sleepmode-loft-sh-ignore-all}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-all: "true"`

**Used on:** Namespace, VirtualClusterInstance

Ignores all activity when determining whether the namespace or vCluster should sleep.

### sleepmode.loft.sh/ignore-ingresses {#sleepmode-loft-sh-ignore-ingresses}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-ingresses: "true"`

**Used on:** Namespace, VirtualClusterInstance

Ignores ingress requests when determining activity. Useful when the namespace receives automated health checks that should not prevent sleep.

### sleepmode.loft.sh/ignore-groups {#sleepmode-loft-sh-ignore-groups}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-groups: "apps,batch"`

**Used on:** Namespace, VirtualClusterInstance

Ignores requests to specific API groups when determining activity. Comma-separated list of API group names.

### sleepmode.loft.sh/ignore-vclusters {#sleepmode-loft-sh-ignore-vclusters}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-vclusters: "true"`

**Used on:** Namespace

Ignores vCluster-related requests when determining namespace activity.

### sleepmode.loft.sh/ignore-resources {#sleepmode-loft-sh-ignore-resources}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-resources: "pods,configmaps"`

**Used on:** Namespace, VirtualClusterInstance

Ignores requests to specific resource types when determining activity. Comma-separated list of resource names.

### sleepmode.loft.sh/ignore-verbs {#sleepmode-loft-sh-ignore-verbs}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-verbs: "get,list,watch"`

**Used on:** Namespace, VirtualClusterInstance

Ignores requests with specific HTTP verbs when determining activity. Comma-separated list.

### sleepmode.loft.sh/ignore-resource-verbs {#sleepmode-loft-sh-ignore-resource-verbs}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-resource-verbs: "pods.core=get list,deployments.apps=get"`

**Used on:** Namespace, VirtualClusterInstance

Ignores specific verb combinations for specific resources. Format: `resource.group=verb1 verb2, resource2.group=verb3`.

### sleepmode.loft.sh/ignore-resource-names {#sleepmode-loft-sh-ignore-resource-names}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-resource-names: "pods.core=monitoring-pod,configmaps.core=config1"`

**Used on:** Namespace, VirtualClusterInstance

Ignores requests to specific named resources. Format: `resource.group=name1 name2`.

### sleepmode.loft.sh/ignore-active-connections {#sleepmode-loft-sh-ignore-active-connections}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-active-connections: "true"`

**Used on:** Namespace, VirtualClusterInstance

Ignores active connections (such as `kubectl exec` or `kubectl port-forward`) when determining whether to sleep. Allows sleep even with open connections.

### sleepmode.loft.sh/ignore-user-agents {#sleepmode-loft-sh-ignore-user-agents}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/ignore-user-agents: "kube-probe/*,prometheus/*"`

**Used on:** Namespace, VirtualClusterInstance

Ignores requests from specific user agents. Supports trailing wildcards. Comma-separated list.

### sleepmode.loft.sh/disable-ingress-wakeup {#sleepmode-loft-sh-disable-ingress-wakeup}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/disable-ingress-wakeup: "true"`

**Used on:** Namespace, VirtualClusterInstance

Disables automatic wakeup from ingress traffic. When set, the namespace or vCluster remains asleep even when receiving ingress requests.

### sleepmode.loft.sh/disable-metrics-tracking {#sleepmode-loft-sh-disable-metrics-tracking}

**Type:** Annotation

**Example:** `sleepmode.loft.sh/disable-metrics-tracking: "true"`

**Used on:** Namespace, VirtualClusterInstance

Disables metrics-based activity tracking. Only API server activity will be tracked.

## Sleep mode status annotations {#sleep-mode-status-annotations}

These annotations are set by the platform to indicate sleep mode status. They are read-only.

### sleepmode.loft.sh/last-activity {#sleepmode-loft-sh-last-activity}

**Type:** Annotation (read-only)

**Example:** `sleepmode.loft.sh/last-activity: "1706745600"`

**Used on:** Namespace, VirtualClusterInstance

Unix timestamp of the last detected activity. Set automatically by the platform.

### sleepmode.loft.sh/sleeping-since {#sleepmode-loft-sh-sleeping-since}

**Type:** Annotation (read-only)

**Example:** `sleepmode.loft.sh/sleeping-since: "1706745600"`

**Used on:** Namespace, VirtualClusterInstance

Unix timestamp of when the namespace or vCluster entered sleep mode. Present only when sleeping.

### sleepmode.loft.sh/sleep-type {#sleepmode-loft-sh-sleep-type}

**Type:** Annotation (read-only)

**Example:** `sleepmode.loft.sh/sleep-type: "inactivitySleep"`

**Used on:** Namespace, VirtualClusterInstance

Indicates how sleep was triggered. Values: `inactivitySleep`, `forcedSleep`, `forcedDurationSleep`, `scheduledSleep`.

### sleepmode.loft.sh/scheduled-sleep {#sleepmode-loft-sh-scheduled-sleep}

**Type:** Annotation (read-only)

**Example:** `sleepmode.loft.sh/scheduled-sleep: "1706832000"`

**Used on:** Namespace, VirtualClusterInstance

Unix timestamp of the next scheduled sleep based on the sleep schedule.

### sleepmode.loft.sh/scheduled-wakeup {#sleepmode-loft-sh-scheduled-wakeup}

**Type:** Annotation (read-only)

**Example:** `sleepmode.loft.sh/scheduled-wakeup: "1706774400"`

**Used on:** Namespace, VirtualClusterInstance

Unix timestamp of the next scheduled wakeup based on the wakeup schedule.

## Platform management annotations {#platform-management-annotations}

These annotations are used internally by the platform for resource management.

### loft.sh/project-role {#loft-sh-project-role}

**Type:** Label

**Example:** `loft.sh/project-role: "true"`

**Used on:** ClusterRole

Marks a ClusterRole as available for use as a project role in vCluster Platform.

### loft.sh/managed {#loft-sh-managed}

**Type:** Label

**Example:** `loft.sh/managed: "true"`

**Used on:** Various resources

Indicates that this resource is managed by vCluster Platform and should not be modified directly.

### loft.sh/non-deletable {#loft-sh-non-deletable}

**Type:** Annotation

**Example:** `loft.sh/non-deletable: "true"`

**Used on:** Various resources

Prevents deletion of this resource through the platform UI or API.

### loft.sh/custom-links {#loft-sh-custom-links}

**Type:** Annotation

**Example:** `loft.sh/custom-links: '[{"title":"Dashboard","url":"https://example.com"}]'`

**Used on:** VirtualClusterInstance, SpaceInstance

Adds custom links to the platform UI for this resource. Value is a JSON array of objects with `title` and `url` fields.

### loft.sh/paused {#loft-sh-paused}

**Type:** Annotation (read-only)

**Example:** `loft.sh/paused: "true"`

**Used on:** Deployment, StatefulSet

Indicates that this workload has been paused (scaled to 0) by sleep mode.

### loft.sh/paused-replicas {#loft-sh-paused-replicas}

**Type:** Annotation (read-only)

**Example:** `loft.sh/paused-replicas: "3"`

**Used on:** Deployment, StatefulSet

Stores the original replica count before the workload was paused. Used to restore replicas on wakeup.

### loft.sh/paused-date {#loft-sh-paused-date}

**Type:** Annotation (read-only)

**Example:** `loft.sh/paused-date: "2024-02-01T10:30:00Z"`

**Used on:** Deployment, StatefulSet

Stores the timestamp when the workload was paused.

### loft.sh/restoring {#loft-sh-restoring}

**Type:** Annotation (read-only)

**Example:** `loft.sh/restoring: "true"`

**Used on:** VirtualClusterInstance

Indicates that a vCluster is currently being restored from a backup.

### loft.sh/version {#loft-sh-version}

**Type:** Annotation

**Example:** `loft.sh/version: "4.0.0"`

**Used on:** Config resources

Stores the platform version that last modified this resource.

**Type:** Annotation

**Example:** `loft.sh/project-secret-name: "database-credentials"`

The name of the project secret this secret was created from.

Specifies the maximum number of nodes that can be provisioned in this pool.

**Type:** Label

**Example:** `vcluster.loft.sh/node-provider: "aws"`

Identifies the static node pool this node belongs to.

## Audit annotations {#audit-annotations}

These annotations appear in audit log events.

### audit.loft.sh/sent-by-agent {#audit-loft-sh-sent-by-agent}

**Type:** Annotation (read-only)

**Example:** `audit.loft.sh/sent-by-agent: "cluster-name"`

**Used on:** Audit events

Identifies audit events that were propagated from a connected cluster's agent rather than originating at the central platform.

## Internal annotations {#internal-annotations}

These annotations are used internally by vCluster and the platform. They should not be modified manually.

### vcluster.loft.sh/object-name {#vcluster-loft-sh-object-name}

**Type:** Annotation (internal)

**Used on:** Synced resources

Stores the original name of the resource in the virtual cluster.

### vcluster.loft.sh/object-namespace {#vcluster-loft-sh-object-namespace}

**Type:** Annotation (internal)

**Used on:** Synced resources

Stores the original namespace of the resource in the virtual cluster.

### vcluster.loft.sh/object-uid {#vcluster-loft-sh-object-uid}

**Type:** Annotation (internal)

**Used on:** Synced resources

Stores the UID of the resource in the virtual cluster.

### vcluster.loft.sh/created {#vcluster-loft-sh-created}

**Type:** Annotation (internal)

**Used on:** Helm releases

Indicates that this release was created by the vCluster CLI.

### vcluster.loft.sh/distro {#vcluster-loft-sh-distro}

**Type:** Label (internal)

**Used on:** vCluster pods

Identifies the Kubernetes distribution running inside the vCluster (k3s, k8s, k0s, eks).
