This page is a Platform-focused entry point to the `vcluster platform` CLI. It groups commands by task and links each one to its full flag reference.

<!-- vale off -->
The canonical, flag-level reference for every command lives in the [vCluster CLI docs](/docs/vcluster/cli/vcluster_platform). This page shows the Platform UI equivalent next to each CLI command.
<!-- vale on -->

## Common tasks and CLI equivalents

### Start or install the platform

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Install Platform](../install/quick-start-guide.mdx) | Not available in the UI. Platform installs with Helm or the CLI. | [`vcluster platform start`](/docs/vcluster/cli/vcluster_platform_start) |

See the [quickstart guide](../install/quick-start-guide.mdx) or the [Helm installation guide](../install/helm.mdx) for full install flows.

### Log in and access keys

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Log in to Platform](../install/quick-start-guide.mdx#login) | Sign in at the Platform URL | [`vcluster platform login`](/docs/vcluster/cli/vcluster_platform_login) |
| Log out of Platform | Avatar menu (bottom of the sidebar) > `Logout` | [`vcluster platform logout`](/docs/vcluster/cli/vcluster_platform_logout) |
| [Create an access key](../administer/authentication/access-keys.mdx#create-an-access-key) | `Users` > select a user > `Create Access Key` > name it in the drawer > `Create` | [`vcluster platform create accesskey`](/docs/vcluster/cli/vcluster_platform_create_accesskey) |

### Connect a cluster

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Register a cluster with Platform](../administer/clusters/connect-cluster.mdx) | `Clusters` > `Connect Cluster via CLI` > follow the drawer's generated command | [`vcluster platform add cluster`](/docs/vcluster/cli/vcluster_platform_add_cluster) |
| Create a local kubeconfig context for a connected cluster | `Clusters` > kebab menu > `Access`, then copy the generated command | [`vcluster platform connect cluster`](/docs/vcluster/cli/vcluster_platform_connect_cluster) |

### Create, list, and delete tenant clusters

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Create a tenant cluster](../use-platform/virtual-clusters/create/create-with-template.mdx) | project dropdown > `Tenant Clusters` > `Create Tenant Cluster` | [`vcluster platform create vcluster`](/docs/vcluster/cli/vcluster_platform_create_vcluster) |
| [Add an existing tenant cluster](../use-platform/virtual-clusters/add-virtual-clusters.mdx) | `Tenant Clusters` > `Add Existing Cluster` > `Add existing tenant cluster` | [`vcluster platform add vcluster`](/docs/vcluster/cli/vcluster_platform_add_vcluster) |
| List tenant clusters | project dropdown > `Tenant Clusters` | [`vcluster platform list vclusters`](/docs/vcluster/cli/vcluster_platform_list_vclusters) |
| Delete a tenant cluster | `Tenant Clusters` > kebab menu (`More actions`) > `Delete` | [`vcluster platform delete vcluster`](/docs/vcluster/cli/vcluster_platform_delete_vcluster) |

### Create, list, and delete namespaces

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Create a namespace](../use-platform/namespaces/create/create-with-template.mdx) | `Projects` > `Namespaces` > `Create Namespace` > optional template > `Create Namespace` | [`vcluster platform create namespace`](/docs/vcluster/cli/vcluster_platform_create_namespace) |
| List namespaces | `Projects` > `Namespaces` | [`vcluster platform list namespaces`](/docs/vcluster/cli/vcluster_platform_list_namespaces) |
| Delete a namespace | `Namespaces` view, hover the row, click the delete icon in the Actions column | [`vcluster platform delete namespace`](/docs/vcluster/cli/vcluster_platform_delete_namespace) |

See [retrieve a namespace kubeconfig](../use-platform/namespaces/retrieve-kube-config.mdx) for related namespace commands.

### Sleep, wake up, and share resources

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Put a namespace to sleep](../use-platform/namespaces/key-features/sleep-mode.mdx#manual--automatic-sleep--wake-up) | `Namespaces` view, hover the row, click `Sleep` | [`vcluster platform sleep namespace`](/docs/vcluster/cli/vcluster_platform_sleep_namespace) |
| [Wake up a namespace](../use-platform/namespaces/key-features/sleep-mode.mdx#manual--automatic-sleep--wake-up) | `Namespaces` view, hover the Status column, click `Wakeup` | [`vcluster platform wakeup namespace`](/docs/vcluster/cli/vcluster_platform_wakeup_namespace) |
| [Put a tenant cluster to sleep](../use-platform/virtual-clusters/key-features/sleep-mode.mdx) | `Tenant Clusters` > select a cluster > ellipsis menu > `Sleep` | [`vcluster platform sleep vcluster`](/docs/vcluster/cli/vcluster_platform_sleep_vcluster) |
| [Wake up a tenant cluster](../use-platform/virtual-clusters/key-features/sleep-mode.mdx) | `Tenant Clusters` > kebab menu (`More actions`) > `Wake Up`, shown only while sleeping | [`vcluster platform wakeup vcluster`](/docs/vcluster/cli/vcluster_platform_wakeup_vcluster) |
| [Share a tenant cluster](../use-platform/virtual-clusters/manage-access.mdx) | `Tenant Clusters` > `Edit` > `Permissions` > `Add Permission` > `Save Changes` | [`vcluster platform share vcluster`](/docs/vcluster/cli/vcluster_platform_share_vcluster) |
| [Share a namespace](../administer/users-permissions/permissions/namespace.mdx) | `Namespaces` > `Edit` > `Permissions` > `Save Changes` | [`vcluster platform share namespace`](/docs/vcluster/cli/vcluster_platform_share_namespace) |

### Backup, destroy, and reset password

| Task | Platform UI | CLI command |
| --- | --- | --- |
| [Back up Platform](../maintenance/backup-restore/backup-restore-platform.mdx) | Not available in the UI, CLI/Helm only | [`vcluster platform backup management`](/docs/vcluster/cli/vcluster_platform_backup_management) |
| [Destroy a Platform instance](../maintenance/uninstall.mdx#delete-the-platform) | Not available in the UI, CLI/Helm only | [`vcluster platform destroy`](/docs/vcluster/cli/vcluster_platform_destroy) |
| [Reset a user's password](../administer/users-permissions/managing-users/reset-password.mdx) | `Users` > select a user > `Reset Password` > confirm > `Copy & Close` | [`vcluster platform reset password`](/docs/vcluster/cli/vcluster_platform_reset_password) |

Resetting the admin password without UI access is CLI/kubectl only, see [reset the admin password](../administer/authentication/reset-admin-password.mdx).
