[Run:ai](https://www.run.ai/) is a GPU orchestration platform that schedules and manages AI and
machine learning workloads across Kubernetes clusters. vCluster is a certified Kubernetes Distribution for hosting Run:ai,
enabling teams to run isolated, multi-tenant AI environments on shared GPU infrastructure.

Compatibility has been verified  with the following versions.


| Component | Version |
|-----------|---------|
| Kubernetes | v1.34 |
| vCluster | v0.31 |
| Run:ai | v2.24 |

## Deployment models

vCluster supports all three deployment models with Run:ai:

| Model | Description | Tenancy mode |
|-------|-------------|--------------|
| **Shared nodes** | Tenants share host cluster nodes with label-based scheduling. Each tenant gets a separate vCluster with its own Kubernetes API. | Soft |
| **Private nodes** | Each tenant gets a dedicated vCluster with auto-provisioned private nodes. | Hard |
| **Standalone** | A single-tenant deployment where Run:ai manages the full node pool within one vCluster. No multi-tenancy overhead. | Hard |

## Installation

The Run:ai certified stack provisions a vCluster together with the NVIDIA GPU Operator and Run:ai
in a single, tested deployment. See the
[certified-stacks repository](https://github.com/loft-sh/certified-stacks) for prerequisites,
configuration options, and step-by-step setup instructions.
