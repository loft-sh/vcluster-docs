[Run:ai](https://www.run.ai/) is a GPU orchestration platform that schedules and manages AI and machine learning workloads across Kubernetes clusters. vCluster Platform is certified with Run:ai, enabling teams to run isolated, multi-tenant AI environments on shared GPU infrastructure.

## Tested on

| Component | Version |
|-----------|-------------------|
| Run:ai | v2.24 |
| Kubernetes | v1.34 |
| vCluster | v0.31 |

## Deployment models

vCluster Platform supports all three deployment models with Run:ai:

| Model | Description |
|-------|-------------|
| **Shared nodes** | Tenants share host cluster nodes with label-based scheduling. Each tenant gets a separate vCluster with its own Kubernetes API.|
| **Private nodes** | Each tenant gets a dedicated vCluster with auto-provisioned private nodes.|
| **Standalone** | A single-tenant deployment where Run:ai manages the full node pool within one vCluster. No multi-tenancy overhead. |

## Installation

Use the [Certified Stacks](./certified-stacks.mdx) to deploy Run:ai on vCluster Platform. The Run:ai certified stack provisions a vCluster together with the NVIDIA GPU Operator and Run:ai in a single, tested deployment.

See the [certified-stacks repository](https://github.com/loft-sh/certified-stacks) for prerequisites, configuration options, and step-by-step setup instructions.
