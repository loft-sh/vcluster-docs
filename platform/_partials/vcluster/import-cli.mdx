When importing a virtual cluster from the vCluster CLI, you will need to provide the name and namespace of the virtual cluster 
along with the name of the project where this virtual cluster needs to be imported. Optionally, you can make use of the `import-name` flag
to specify a name for this virtual cluster, when it gets displayed under the <NavStep>Virtual Clusters</NavStep> view for the specified project in the vCluster Platform.
```bash
vcluster platform add vcluster [vcluster-name] \
  --namespace [vcluster-namespace] \
  --project [project-name] \
  --import-name [import-name]
```
The above command will search for the virtual cluster in the current context. 
If you want to import a virtual cluster from another connected cluster, you can use the `--context` flag to specify the context of the 
cluster where the given virtual cluster is present.
```bash
vcluster platform add vcluster [vcluster-name] \
  --namespace [vcluster-namespace] \
  --project [project-name] \
  --import-name [import-name] \
  --context [context]
```

:::tip Kube-Context
In case the cluster where the virtual cluster exists is not present in your kubeconfig, you can run: 
```bash
vcluster platform connect cluster [cluster-name]
```
to first connect to that cluster. This will set your current context automatically.

You can then run the first command above to import your desired virtual cluster. Do remember to run `vcluster disconnect` to go back to your previous context.
:::
