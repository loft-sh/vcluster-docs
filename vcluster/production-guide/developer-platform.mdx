Provide self-service Kubernetes clusters for development teams, CI/CD pipelines, and ephemeral environments. Teams create and delete clusters on demand. Shared nodes keep cost and complexity low. Platform enforces quotas and policies.

**Typical stack:** Platform on an existing Kubernetes cluster. Shared node pool. Optional dedicated node pools for integration or performance-sensitive workloads.

**What makes this path different:** Cluster lifecycle is short and frequent. The value is self-service speed and governance, not hardware isolation.

## Day 0: Design decisions

| Decision | Read next | Outcome |
| --- | --- | --- |
| Define cluster lifecycle policy | [Templates](/platform/administer/templates/overview), [sleep mode](../configure/vcluster-yaml/sleep.mdx) | Set default cluster TTL, sleep schedules, and auto-deletion for ephemeral clusters. |
| Define team quotas | [Quotas](/platform/administer/projects/quotas), [resource quota](../configure/vcluster-yaml/policies/resource-quota.mdx) | Set per-team cluster count, CPU, and memory limits. |
| Plan sync configuration | [Sync configuration](../configure/vcluster-yaml/sync/overview.mdx) | Decide which control plane cluster resources (storage classes, ingress controllers, secrets) are visible inside tenant clusters. |
| Plan CI/CD integration | [vCluster CLI](../cli/vcluster_create.md), [Argo CD](/platform/integrations/argocd) | Choose how pipelines create and destroy ephemeral clusters: CLI, Terraform, Argo CD, or Platform API. |

## Day 1: Stand up the platform

1. Install [vCluster Platform](/platform/install/quick-start-guide) on your existing Kubernetes cluster.
2. Configure [SSO](/platform/configure/platform-configs/single-sign-on) and [team permissions](/platform/administer/users-permissions/overview).
3. Create [projects](/platform/administer/projects/create), [templates](/platform/administer/templates/overview), and [quotas](/platform/administer/projects/quotas) per team.
4. Set shared node pool policies: [pod security standard](../configure/vcluster-yaml/policies/pod-security-standard.mdx), [network policy](../configure/vcluster-yaml/policies/network-policy.mdx), and [resource quota](../configure/vcluster-yaml/policies/resource-quota.mdx).
5. Configure cluster sleep and deletion policies in templates.
6. Integrate with [Argo CD](/platform/integrations/argocd) or CI tooling for automated cluster provisioning in pipelines.
7. Validate self-service: a developer creates a cluster, deploys a workload, and deletes the cluster with no ops intervention required.
