### Reuse host metrics server

Create a `vcluster.yaml` with:

```yaml
integrations:
  metricsServer:
    enabled: true
    nodes: true
    pods: true
```

Then create a vCluster with:

```
vcluster create host-metrics-vcluster -f vcluster.yaml
```

Then the metrics api should be available within the vCluster. This will obviously require a metrics server to be installed on the host cluster.

If your metrics server is not using the default `kube-system/metrics-server` service, you can also configure a custom one via:

```yaml
integrations:
  metricsServer:
    enabled: true
    apiService:
      service:
        name: my-custom-metrics-server
        namespace: my-custom-namespace
        port: 443
```

If you are using [network policies](/docs/vcluster/configure/vcluster-yaml/policies/network-policy), you need to add the metrics server pod to the allowed rules that the control plane can communicate with:

```yaml
policies:
  networkPolicy:
    controlPlane:
      egress:
        - to:
            - namespaceSelector:
                matchLabels:
                  kubernetes.io/metadata.name: 'kube-system'
              podSelector:
                matchLabels:
                  k8s-app: metrics-server
```
