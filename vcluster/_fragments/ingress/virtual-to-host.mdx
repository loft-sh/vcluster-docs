You can create an ingress in a tenant cluster to make a service in that tenant cluster available via a hostname/domain. Instead of having to run a separate ingress controller in each tenant cluster, you can sync the ingress resource to the underlying control plane cluster, which means that the tenant cluster can use a shared ingress controller that is running in the control plane cluster. 

```yaml
sync:
  toHost:
    ingresses:
      enabled: true
```

Because the syncer keeps typical SSL provisioning-related annotations for ingresses, you can also set the cert-manager ingress annotations on an ingress in your tenant cluster to use the cert-manager of the underlying control plane cluster to automatically provision SSL certificates from Let's Encrypt.
