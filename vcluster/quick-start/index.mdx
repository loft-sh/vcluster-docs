Two questions determine which quickstart is right for you.

**Do you already have a Kubernetes cluster?**

- **Yes** — your existing cluster becomes the Control Plane Cluster. Choose how tenant workloads run: on shared infrastructure (OSS vCluster CLI) or on dedicated nodes per tenant (requires vCluster Platform).
- **No, I need local dev or CI** — use vind to run a complete cluster in Docker with no Kubernetes dependency.
- **No, I need a production Control Plane Cluster** — use vCluster Standalone to bootstrap one from bare metal or VMs, then install Platform on it.

**How should tenant workloads run?**

- **Shared nodes** — tenant workloads run on the existing cluster's node pool. Works with the OSS vCluster CLI. Fastest to set up. Suited for developer platforms and CI pipelines.
- **Private nodes** — each tenant cluster gets dedicated worker nodes. Full infrastructure isolation for GPU workloads and regulated environments. Requires vCluster Platform (free mode supported).
