Use the `vcluster.yaml` file to configure vCluster and override default settings. If you're familiar with [Helm](https://helm.sh/), you can use `vcluster.yaml` in the same way as a `values.yaml` file. All vCluster deployment methods use Helm under the hood, so the behavior is consistent.

The configuration file offers advantages over CLI flags. It provides access to all vCluster settings (CLI flags only expose common options), enables version control for tracking configuration changes, allows you to reuse the same configuration across multiple environments, and facilitates team collaboration through shared configuration files.

To review the default configuration, see the [vCluster chart values file](https://github.com/loft-sh/vcluster/blob/main/chart/values.yaml). The [vCluster Helm chart](https://github.com/loft-sh/vcluster/blob/main/chart/values.schema.json) also includes a JSON schema for validating `vcluster.yaml`. For more information on configuration structure, see the [What is vcluster.yaml? documentation](/vcluster/configure/vcluster-yaml/what-is-vcluster-yaml).

## Create a virtual cluster with a config file

:::warning
vCluster 0.20.0 introduced the `vcluster.yaml` format, which replaces the older `values.yaml` file used in versions before 0.20.0. If you're upgrading from vCluster 0.19.x or earlier, use the [conversion guide](../../reference/migrations/0-20-migration) to automatically convert your existing `values.yaml` file to the new format.
:::

Configure your vCluster installation in an optional `vcluster.yaml` configuration file. Then deploy your changes.
