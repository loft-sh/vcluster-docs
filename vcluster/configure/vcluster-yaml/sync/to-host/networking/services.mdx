By default, this is enabled.

Sync all [Service](https://kubernetes.io/docs/concepts/services-networking/service/) resources from the tenant cluster to the control plane cluster.

:::warning LoadBalancer address pool governance on shared nodes
A synced Service of type `LoadBalancer` draws its address from whatever pool your load balancer controller (for example, MetalLB) makes available on the control plane cluster. Without pool scoping, a tenant can claim an address from any pool, including a management-network or public pool. Scope address pools with namespace or Service selectors, or gate pool and `loadBalancerIP` selection with admission policy, and treat public IP exposure as an explicit opt-in rather than an automatic default. See [network and endpoint isolation](../../../../../security/shared-nodes-hardening.mdx#network-and-endpoint-isolation).
:::

## Patches

Use `sync.toHost.services.patches` to transform Service fields, such as `spec.ports[*].name`, while syncing to the control plane cluster. See [Patching synced resources](../../patching/README.mdx) for syntax, directionality, and examples.
