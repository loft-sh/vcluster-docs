Before deploying a vCluster, it's important to understand the components of a tenant cluster and how they determine your isolation model.

- [**Architecture of Tenancy Models**](../../introduction/architecture.mdx): Different tenancy models can be achieved based on how vCluster is built, but the main two components that determine your isolation model are the virtualized control plane and worker nodes.
- [**Virtualized Control Plane**](../../introduction/architecture.mdx#virtual-control-plane): Contains a Kubernetes API server, a controller manager, and a data store. Runs on the Control Plane Cluster but is completely invisible to tenants.
- [**Worker Nodes**](../../configure/tenancy-model.mdx): Worker nodes can be drawn from the Control Plane Cluster's node pool or provisioned as dedicated private nodes.
