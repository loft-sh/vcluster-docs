:::warning Shared nodes are for trusted tenants
Shared nodes give each tenant control-plane, API, and namespace isolation, but tenant workloads share the same kernel and physical nodes. They're a supported boundary for trusted tenants, not a security boundary for untrusted tenants with Kubernetes access or arbitrary workload execution. Good fits are development, testing, CI/CD, and internal engineering teams.

Use [private nodes](/docs/vcluster/deploy/worker-nodes/private-nodes) for external, resale, regulated, or otherwise untrusted tenant offerings, optionally with [vNode](https://www.vnode.com/docs/) for runtime isolation.

Even for permitted use, shared-node isolation depends on controls that must be actively enforced. Some CNIs accept NetworkPolicy resources without enforcing them, which silently removes network isolation. Verify your control plane cluster's CNI and apply the [security baseline](/docs/vcluster/security) before you rely on shared-node isolation.
:::
