:::warning Shared nodes are for internal, trusted tenants
Shared nodes give each tenant API and namespace isolation, but tenant workloads share the same kernel and physical nodes. Use shared nodes only for internal, trusted tenants. Good fits are development, testing, CI/CD, and internal engineering teams.

Don't use shared nodes for externally facing or resold tenant offerings. For untrusted or external tenants, use [private nodes](/docs/vcluster/deploy/worker-nodes/private-nodes) for infrastructure level separation, optionally with [vNode](https://www.vnode.com/docs/) for runtime isolation.

Even for permitted internal use, shared-node isolation depends on controls that must be actively enforced. Some CNIs accept NetworkPolicy resources without enforcing them, which silently removes network isolation. Verify your control plane cluster's CNI and apply the [security baseline](/docs/vcluster/security) before you rely on shared-node isolation.
:::
