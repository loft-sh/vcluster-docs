In general, you don't need to adjust RBAC rules. Most are automatically generated by [enabling resource syncing](/vcluster/vcluster-yaml/sync/). One reason to configure RBAC might be to grant vCluster access to existing resources. For example, you might want vCluster to use an existing ServiceAccount resource instead of creating a new one.

:::note
When enabled, the following config fields may adjust default RBAC roles:

- [experimental.multiNamespaceMode](/vcluster/vcluster-yaml/experimental/multi-namespace-mode.mdx)
- [experimental.genericSync](/vcluster/vcluster-yaml/experimental/generic-sync.mdx)
- [plugins](/vcluster/vcluster-yaml/plugins.mdx)
:::
