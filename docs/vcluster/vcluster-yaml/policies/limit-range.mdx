You can create a [LimitRange](https://kubernetes.io/docs/concepts/policy/limit-range/) policy to constrain virtual cluster resource allocation. vCluster creates the LimitRange on the host in the same namespace as vCluster itself. LimitRange only applies to synced resources, such as pods.

## Example

If you would normally deploy the following LimitRange:

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: vcluster-limit-range
spec:
  limits:
  - default:
      memory: 512Mi
      cpu: "1"
    defaultRequest:
      memory: 128Mi
      cpu: 100m
    type: Container
```

Instead configure the LimitRange in your `vcluster.yaml` config file like this:

```yaml
limitRange:
  enabled: true
  default:
    memory: 512Mi
    cpu: "1"
  defaultRequest:
    memory: 128Mi
    cpu: 100m
```
