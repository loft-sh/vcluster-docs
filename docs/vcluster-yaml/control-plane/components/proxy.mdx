- Advanced options to set the virtual cluster's API server's bindAddress & port
- These settings control the address & port that the API server listens on, and for authentication purposes they are also used when generating the certificate for TLS
- `proxy.bindAddress` was previously the `--bind-address` flag in the previous [configuration reference](https://www.vcluster.com/docs/config-reference)
- `proxy.port` was previously the `--port` flag in the previous [configuration reference](https://www.vcluster.com/docs/config-reference)
- `proxy.extraSANs` was previously the `--tls-san` flag in the previous [configuration reference](https://www.vcluster.com/docs/config-reference)
